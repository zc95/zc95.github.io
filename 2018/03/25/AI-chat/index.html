遇到的坑

坑

本地打开html文件，能正常访问。但放到github上之后，访问 https://zc95.github.io/demo/AI-Chat 就会报错。

谷歌：

Failed to load file://www.tuling123.com/openapi/api?key=xxxxxx: Cross origin requests are only supported for protocol schemes: http, data, chrome, chrome-extension, https.

Safari：

XMLHttpRequest cannot load http://www.tuling123.com/openapi/api?key=xxxxxx due to access control checks.

原因

​ 这个智能语音是用的 图灵智能机器人 的接口，但是我注意到他们的url是http协议的，而github是https协议的，因此初步猜测和这个有关，后来我把页面挂在一个http协议的开源网站jsrun上面，发现确实和猜测的一样了。

​ HTTPS页面里动态的引入HTTP资源，比如引入一个js文件，会被直接block掉的，在HTTPS页面里通过AJAX的方式请求HTTP资源，也会被直接block掉的。这是因为浏览器为了安全，HTTPS下不能跨协议直接调用http。

解决方案

方法1：对于同时支持HTTPS和HTTP的资源，引用的时候要把引用资源的URL里的协议头去掉，浏览器会自动根据当前是HTTPS还是HTTP来给资源URL补上协议头的，可以达到无缝切换。或者在head中添加meta标签，会自动将http的不安全请求升级为https。

方法2：使用iframe的方式引入HTTP资源，然后将这个页面嵌入到HTTPS页面里就可以了。

方法3：由于该智能聊天的url请求只能是http协议，所以不采用上面两种方法。当跨域请求被拒绝时，地址栏右侧会有个盾牌，点击 加载不安全的脚本 ，允许网页加载这些“不安全”的脚本或请求，当然这是建立在访问双方互相信任的基础上，如果涉及银行卡，密码之类的最好不要这么做。

此时，接口已经能正常访问，地址栏前面的绿色安全变成红色不安全，控制台也从报错变为警告。
